Northwestern Formula SAE:
Data Acquisition System

Embedded software and PCB design for Northwestern Formula Racing's DAQ subteam — contributing to a ~25,000-line C++ codebase alongside a team of 7, developing dashboard firmware, controller hardware, and tooling for a Formula SAE race car.

C++ Embedded Software PCB Layout Altium Designer CAN Bus Python Git

1. Overview

The Project

Northwestern Formula Racing's Data Acquisition subteam builds the embedded systems behind the car's telemetry, driver interface, and real-time vehicle monitoring. I joined a team of 7 working on a codebase of roughly 25,000 lines of C++ (plus ~10,000 lines across other languages) — a large, active project that required getting up to speed quickly and contributing meaningfully from early on.

My work spans both hardware and software: PCB layout and bring-up of the dashboard controller, application-layer firmware for driver feedback, and tooling for integration testing and debugging across the DAQ and low-voltage electronics systems.

2. Hardware

Dashboard Controller PCB

I laid out V2 and contributed to V3 of the dashboard controller PCB in Altium Designer, iterating on the initial design to address issues found during bring-up. The layout involved careful handling of polygon pours, higher-current power sections, and differential signal routing within the car's packaging constraints.

SubsystemDetails
Polygon PoursGround and power planes with deliberate pour strategy for signal integrity and thermal management
High-Current SectionsTrace width and copper weight sizing for power delivery to display and LED subsystems
Differential SignalsLength-matched differential pairs for CAN bus communication reliability
CAN TransceiverInterface to vehicle CAN network for telemetry and ECU communication
Display InterfaceDriver-facing readout for real-time vehicle state and fault conditions

After layout, I handled full bring-up: hand-soldering components, debugging electrical issues, and running integration tests to validate each subsystem before vehicle installation.

3. Embedded Software

Firmware & Tooling

The dashboard firmware is written in C++ and architected to parse incoming CAN messages and surface meaningful vehicle state to the driver. A key design goal was fast iteration — the architecture allowed new feedback features to be implemented quickly as requirements evolved across the season.

  • Wrote application-layer firmware giving the driver real-time insight into vehicle state, battery health, braking, and system faults via display and NeoPixel LED indicators
  • Architected the display system to allow rapid implementation of new screens as driver and team feedback came in throughout the season
  • Built dedicated debugging screens for integration testing of the DAQ system and other low-voltage electronics — used during bringup and track testing
  • Wrote a Python script to auto-generate debugging screen code, cutting manual screen-definition time significantly
  • Maintained CAN communication software as DBC definitions evolved, keeping the dashboard in sync with other vehicle systems
  • Onboarded into a ~25,000-line C++ codebase as an underclassman and made meaningful contributions within the first weeks
