Northwestern Formula SAE:
Data Acquisition System

Building the systems that collect, display, and interpret real-time vehicle data for Northwestern's Formula SAE race car — from dashboard firmware to custom PCB design.

C++ PCB Design Altium Designer Embedded Systems CAN Bus Git

1. Overview

The Project

Northwestern Formula Racing's Data Acquisition subteam develops the systems responsible for collecting, transmitting, and visualizing vehicle telemetry. For NFR26, the team is redesigning the race car's dashboard system, which provides drivers with real-time information about vehicle state, battery health, braking, steering, and system faults during operation.

My work focuses on both the hardware and software behind the dashboard. I've contributed to the redesign and validation of the dashboard controller hardware, assembled and brought up multiple PCB revisions, and developed software features for driver feedback, telemetry visualization, and CAN network debugging.

2. Hardware

The PCB

I designed Version 2 of the dashboard controller PCB in Altium Designer, improving on the previous iteration with better component placement and layout for the car's packaging constraints.

3. Software

Firmware & Tools

The dashboard firmware is written in C++ and responsible for parsing incoming CAN messages, updating the display, and driving the LED indicators based on vehicle state.

  • Validated and brought up dashboard controller hardware, including CAN interfaces, displays, and driver I/O peripherals
  • Assembled and tested multiple dashboard controller PCB revisions prior to vehicle integration
  • Developed NeoPixel drivers and animations for driver feedback and vehicle status indication
  • Implemented dashboard debugging and telemetry screens for vehicle data visualization
  • Maintained and updated CAN communication software as DBC definitions evolved throughout the season
  • Created Python tools to streamline testing, debugging, and development workflows
