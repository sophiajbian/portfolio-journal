Resonance —
Violin Posture Monitoring System

A shoulder-rest attachment that tracks violin orientation in real time using a 9-DoF IMU and streams data to a browser-based companion app — helping violinists observe postural drift during practice without interrupting their playing.

Embedded Systems IMU / Sensors RP2040 3D Printing Web App WebSerial API User Research

Drop resonance.jpg into the portfolio-journal folder to add a photo

1. The Problem

Background

Violinists hold their instruments between chin and shoulder for hours at a time — an asymmetric position that can cause neck, jaw, shoulder, and back pain. From interviews with 27 violinists, we found that many believed their discomfort stemmed from poor playing posture, and that players practicing 4+ hours per day reported higher pain levels than those who practiced less.

The challenge is that postural drift happens gradually: as a musician fatigues, their position shifts in ways that are subtle enough to go unnoticed mid-practice. Resonance was built to make these changes visible in real time — as a posture-awareness tool, not a medical device.

2. Hardware

The Device

The system consists of a compact, 3D-printed housing (2.25 × 1.19 × 0.25 in, 18g total) that attaches to the violin's shoulder rest via removable adhesive — no permanent modifications. Inside sits an IMU and microcontroller, stacked to minimize footprint.

ComponentDetails
MCUAdafruit Feather RP2040 — 2.0 × 0.9 × 0.28 in, includes rechargeable battery for extended sessions
IMUAdafruit LSM6DSOX + LIS3MDL 9-DoF — accelerometer, gyroscope, and magnetometer; magnetometer prevents gyroscope drift accumulation over long sessions
HousingCustom 3D-printed enclosure; 18g with all components; attaches via removable adhesive strip compatible with any shoulder rest
ConnectionUSB-C to host computer; streams at ~100 Hz; pitch, roll, yaw + linear acceleration + angular velocity per packet

Multiple housing iterations were designed and tested for fit, stability, and component placement. The magnetometer was specifically chosen to correct for gyroscope drift that would accumulate over a multi-hour practice session.

3. Software & Interface

Companion Web App

The browser reads the sensor stream directly via the WebSerial API — no install, driver, or background service required. Opening the page and plugging in the device is the entire setup. At the start of each session, the player holds their playing position for a few seconds to set a neutral reference; all subsequent measurements are compared against that reference.

  • Live view: A 3D render of the shoulder rest rotates in real time to mirror the violin's orientation; the offending axis tints red and a banner ("Scroll dipping — raise neck") appears when posture crosses a threshold
  • Posture score: Computed from how closely the instrument remains to the neutral reference over time; smoothed over a rolling window so brief movements don't disproportionately penalize the score
  • Post-session dashboard: Shows score, session duration, alert count, and which axis (pitch, roll, or yaw) the player spent the most time out of range — surfaced as a focus for the next session
  • Session history: Scrollable list of all past sessions with full sensor traces; optional cloud sync for sharing data with teachers or tracking across devices
  • Accessibility: Visual-only feedback — no audio — so alerts register in peripheral vision without competing with the sound of the instrument
